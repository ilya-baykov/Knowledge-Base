# DRF Serializers — Шпаргалка

---

## Что такое сериализатор и зачем он нужен?

Django ORM возвращает объекты Python. HTTP-ответ — это текст (обычно JSON).
Сериализатор — это мост между ними в **обе стороны**:

```
Python-объект  →  [Serializer]  →  JSON  (сериализация, для ответа)
JSON / form-data  →  [Serializer]  →  Python-объект  (десериализация, для сохранения)
```

Дополнительно сериализатор берёт на себя **валидацию** входящих данных —
то, что в обычном Django делают формы (Forms).

Аналогия: если `Model` — это структура БД, то `Serializer` — это то,
что определяет, **какие поля видны снаружи и как они выглядят**.

---

## Основные типы сериализаторов

### 1. `Serializer` — базовый

Полностью ручной. Поля описываются вручную, логика сохранения пишется вручную.

```python
from rest_framework import serializers

class ArticleSerializer(serializers.Serializer):
    id    = serializers.IntegerField(read_only=True)
    title = serializers.CharField(max_length=200)
    body  = serializers.CharField()
    published_at = serializers.DateTimeField(read_only=True)

    def create(self, validated_data):
        return Article.objects.create(**validated_data)

    def update(self, instance, validated_data):
        instance.title = validated_data.get('title', instance.title)
        instance.body  = validated_data.get('body', instance.body)
        instance.save()
        return instance
```

**Когда использовать:** данные не привязаны к модели (расчёты, агрегаты,
внешние API, сложные трансформации).

---

### 2. `ModelSerializer` — автоматический

Берёт поля из модели сам. `create()` и `update()` реализованы по умолчанию.

```python
from rest_framework import serializers
from .models import Article

class ArticleSerializer(serializers.ModelSerializer):
    class Meta:
        model  = Article
        fields = ['id', 'title', 'body', 'published_at']
        # или fields = '__all__'
        read_only_fields = ['id', 'published_at']
```

**Когда использовать:** стандартный CRUD поверх Django-модели — это 90% случаев.

---

### 3. `HyperlinkedModelSerializer` — со ссылками вместо ID

Вместо числового `id` использует URL (`"url": "http://api/articles/5/"`)

```python
class ArticleSerializer(serializers.HyperlinkedModelSerializer):
    class Meta:
        model  = Article
        fields = ['url', 'title', 'body']
```

**Когда использовать:** REST API в стиле HATEOAS, когда хочется,
чтобы клиент навигировал по ссылкам, а не знал структуру URL заранее.
На практике используется редко — чаще просто возвращают `id`.

---

### 4. `ListSerializer` — для списков

Используется неявно, когда передаёшь `many=True`.
Можно переопределить для кастомной bulk-логики.

```python
# Обычное использование (many=True создаёт ListSerializer автоматически)
serializer = ArticleSerializer(queryset, many=True)

# Кастомный ListSerializer для bulk create/update
class ArticleBulkSerializer(serializers.ListSerializer):
    def create(self, validated_data):
        articles = [Article(**item) for item in validated_data]
        return Article.objects.bulk_create(articles)

class ArticleSerializer(serializers.ModelSerializer):
    class Meta:
        model = Article
        fields = ['title', 'body']
        list_serializer_class = ArticleBulkSerializer
```

## Ключевые возможности сериализаторов

### Валидация

```python
class ArticleSerializer(serializers.ModelSerializer):
    class Meta:
        model  = Article
        fields = ['title', 'body']

    # Валидация одного поля
    def validate_title(self, value):
        if 'spam' in value.lower():
            raise serializers.ValidationError("Спам не принимается.")
        return value

    # Валидация нескольких полей вместе
    def validate(self, attrs):
        if attrs['start_date'] > attrs['end_date']:
            raise serializers.ValidationError("Дата начала позже даты конца.")
        return attrs
```

---

### Кастомные поля — `SerializerMethodField`

Поле только для чтения, значение которого вычисляется методом.

```python
class ArticleSerializer(serializers.ModelSerializer):
    word_count    = serializers.SerializerMethodField()
    author_name   = serializers.SerializerMethodField()

    class Meta:
        model  = Article
        fields = ['title', 'word_count', 'author_name']

    def get_word_count(self, obj):
        return len(obj.body.split())

    def get_author_name(self, obj):
        return obj.author.get_full_name()
```

---

### Вложенные сериализаторы

```python
class AuthorSerializer(serializers.ModelSerializer):
    class Meta:
        model  = User
        fields = ['id', 'username', 'email']

class ArticleSerializer(serializers.ModelSerializer):
    author = AuthorSerializer(read_only=True)   # вложенный объект

    class Meta:
        model  = Article
        fields = ['id', 'title', 'author']
```

> ⚠️ Вложенные сериализаторы по умолчанию `read_only`.
> Если нужна запись через вложенный объект — придётся переопределять `create/update` вручную.

---

### `source` — переименование полей и обращение к связанным данным

```python
class ArticleSerializer(serializers.ModelSerializer):
    # Переименовать поле
    heading = serializers.CharField(source='title')

    # Поле из связанной модели (через точку)
    author_email = serializers.EmailField(source='author.email', read_only=True)

    class Meta:
        model  = Article
        fields = ['heading', 'author_email']
```

---

### `extra_kwargs` — настройка полей без явного объявления

```python
class ArticleSerializer(serializers.ModelSerializer):
    class Meta:
        model  = Article
        fields = ['id', 'title', 'body', 'password']
        extra_kwargs = {
            'password': {'write_only': True},
            'title':    {'min_length': 5},
        }
```

## Сравнение: `Serializer` vs `ModelSerializer`

| Критерий              | `Serializer`          | `ModelSerializer`           |
|-----------------------|-----------------------|-----------------------------|
| Поля                  | Описываются вручную   | Берутся из модели            |
| `create/update`       | Пишешь сам            | Реализованы автоматически    |
| Гибкость              | Максимальная          | Достаточная для 90% случаев  |
| Код                   | Больше                | Меньше                      |
| Привязка к модели     | Нет                   | Обязательна                 |
| Когда выбирать        | Нет модели / сложная логика | Стандартный CRUD       |

## Кейсы применения

### Кейс 1: Разные сериализаторы для чтения и записи

Частый паттерн — при записи принимать `id`, при чтении отдавать полный объект.

```python
# При создании статьи принимаем author как ID
class ArticleWriteSerializer(serializers.ModelSerializer):
    class Meta:
        model  = Article
        fields = ['title', 'body', 'author']  # author — FK, принимает ID

# При чтении отдаём вложенный объект автора
class ArticleReadSerializer(serializers.ModelSerializer):
    author = AuthorSerializer()

    class Meta:
        model  = Article
        fields = ['id', 'title', 'body', 'author', 'published_at']


# Во view выбираем нужный:
def get_serializer_class(self):
    if self.request.method in ('POST', 'PUT', 'PATCH'):
        return ArticleWriteSerializer
    return ArticleReadSerializer
```

---

### Кейс 2: Сериализатор без модели (например, форма логина)

```python
class LoginSerializer(serializers.Serializer):
    username = serializers.CharField()
    password = serializers.CharField(write_only=True)

    def validate(self, attrs):
        user = authenticate(**attrs)
        if not user:
            raise serializers.ValidationError("Неверные учётные данные.")
        attrs['user'] = user
        return attrs
```

---

### Кейс 3: Bulk create через `many=True`

```python
# Один и тот же сериализатор работает для одного объекта и для списка
data = [{'title': 'A', 'body': '...'}, {'title': 'B', 'body': '...'}]
serializer = ArticleSerializer(data=data, many=True)
if serializer.is_valid():
    serializer.save()
```

---

### Кейс 4: Переопределение `create` для сложной логики

```python
class RegisterSerializer(serializers.ModelSerializer):
    password  = serializers.CharField(write_only=True)
    password2 = serializers.CharField(write_only=True)

    class Meta:
        model  = User
        fields = ['username', 'email', 'password', 'password2']

    def validate(self, attrs):
        if attrs['password'] != attrs['password2']:
            raise serializers.ValidationError("Пароли не совпадают.")
        return attrs

    def create(self, validated_data):
        validated_data.pop('password2')
        user = User.objects.create_user(**validated_data)
        return user
```

## Важные атрибуты полей — шпаргалка

| Атрибут        | Что делает                                              |
|----------------|---------------------------------------------------------|
| `read_only`    | Поле только в ответе, игнорируется при записи           |
| `write_only`   | Поле только при записи, не попадает в ответ (пароли)   |
| `required`     | Обязательно при записи (по умолчанию `True`)            |
| `allow_null`   | Разрешает `null`                                        |
| `default`      | Значение по умолчанию, если поле не передано            |
| `source`       | Откуда брать значение (имя атрибута или путь через `.`) |
| `validators`   | Список дополнительных валидаторов                       |

## Жизненный цикл сериализатора

```
# ЧТЕНИЕ (сериализация)
serializer = ArticleSerializer(instance)   # передаём объект
serializer.data                            # → OrderedDict / JSON-ready dict

# ЗАПИСЬ (десериализация)
serializer = ArticleSerializer(data=request.data)
serializer.is_valid(raise_exception=True)  # валидация, бросает 400 при ошибке
serializer.validated_data                  # очищенные данные
serializer.save()                          # вызывает create() или update()

# ОБНОВЛЕНИЕ (partial)
serializer = ArticleSerializer(instance, data=request.data, partial=True)
```
